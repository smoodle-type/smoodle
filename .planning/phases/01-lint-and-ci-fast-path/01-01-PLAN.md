# Schema YAML structure summary (from schema/thai_phonetic.schema.yaml)

Top-level keys present in v0.0.6:
- `schema:` (with sub-keys `schema_id`, `name`, `version`, `author`, `description`)
- `switches:` (list)
- `engine:` (with sub-keys `processors`, `segmentors`, `translators`, `filters`)
- `speller:` (with sub-keys `alphabet`, `delimiter`, `algebra`)
- `translator:` (with sub-keys including `dictionary`, `enable_completion`, `enable_sentence`, `initial_quality`)
- Optional Rime-defined: `punctuator`, `key_binder`, `recognizer`

Required schema.* sub-keys: `schema_id` (string), `name` (string), `version` (quoted string).

Algebra rules: list of strings under `speller.algebra`, each of the shape
`<op>/<pattern>/<replacement>/[<flags>]` where `<op>` ∈ {xform, derive, xlit, erase, abbrev, fuzz}.
The v0.0.6 schema uses only `derive/`. Each rule must have at least 3 slash-separated parts
(op, pattern, replacement) — the trailing slash counts as the empty 4th part.

# Dict YAML structure summary (from schema/thai_phonetic.dict.yaml)

Two-document format. Frontmatter (terminated by `...`):
- `name` (string, must equal `schema_id`)
- `version` (quoted string, must equal `schema.version`)
- `sort` (`by_weight`)
- `use_preset_vocabulary` (bool, currently `false`)
- `columns` (list — must contain `text`, `code`, `weight`)
- `encoder` (object, optional)

Body (TSV after `...`): one row per line, exactly tab-separated, format `<thai>\t<romanization>\t<weight>`.
Weight column: must be a non-negative integer (positive integer in current data; zero theoretically permitted).
PyYAML cannot parse the body cleanly — parse line-by-line, see tests/test_dict.py:94-125 and scripts/merge_dict.py:106-117 for the canonical pattern.

# default.custom.yaml structure (from schema/default.custom.yaml)

```yaml
patch:
  schema_list:
    - schema: thai_phonetic
    - schema: luna_pinyin
    # ...
```

Top-level key `patch:` only. Its sub-keys are arbitrary Rime config paths to override (here only `schema_list`).

# import_preset semantics

Rime's `import_preset` directive (commonly under `key_binder`, `punctuator`, `recognizer`) takes a string
naming a preset YAML in the same directory. Valid presets shipped with Rime/Squirrel that the lint must
treat as resolvable: `default`, `symbols`, `key_bindings`, `punctuation`. The v0.0.6 smoodle schema does
NOT currently use `import_preset` — but the lint must still validate the reference if it appears (i.e.,
the negative fixture `broken_schema_bad_import_preset.yaml` introduces one with an unknown preset name).

# .yamllint baseline

The v0.0.6 schema files are written by hand. They include:
- 2-space indentation throughout
- Lines longer than 80 chars in the long descriptive comment block at the top of thai_phonetic.schema.yaml
- A truncation header in thai_phonetic.dict.yaml around the two-document `---`/`...` boundary

The .yamllint config must accept all three files as-is. Use `extends: default` and override the rules
that would reject the baseline (specifically: `line-length: disable` because dict body has long lines,
`document-start: disable` because dict file uses `---` mid-file not at top, and possibly relax
`indentation` if needed — verify by running yamllint locally before declaring done).
