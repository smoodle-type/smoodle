# v0.0.8b manual e2e checklist

Founder smoke before flipping 8b-C3/C4/C5 PASS. Run on a fresh-ish Mac
(or wipe `~/Library/Rime/` first if reusing dogfood machine).

## Prep

1. Download fresh DMG:
   ```
   curl -L -o /tmp/Smoodle-v0.0.8b.dmg https://github.com/smoodle-type/smoodle-app/releases/download/v0.0.8b/Smoodle-v0.0.8b-universal.dmg
   open /tmp/Smoodle-v0.0.8b.dmg
   ```

2. Drag-install both apps from the DMG mount:
   - `Smoodle.app` → Input Methods shortcut (auths into `/Library/Input Methods/`)
   - `Smoodle Config.app` → `/Applications/`

3. Right-click → Open each one to clear Gatekeeper. Confirm both launch
   without "damaged / can't be opened" warnings.

4. System Settings → Keyboard → Input Sources → Add → Thai → Smoodle.
   Switch active input source to Smoodle.

## Forget (8b-C6) auxiliary smoke

After row 10 (telemetry off), if you have a saved install_id_hash:
```
cat ~/.smoodle/install_id      # the hash
curl -X DELETE "https://forget.0dl.me/api/forget?install_id_hash=<hash>" \
  -H "Authorization: Bearer $(cat ~/.smoodle/forget_token)"
```
Expect `{"deleted": N}`. Then verify Config.app Delete-my-data button
reproduces same deletion count when called against a fresh hash.

## Known v0.0.8b limitations (not blockers)

- "app" binary name inside `Smoodle Config.app/Contents/MacOS/app` (Cargo
  package name is `app`, Tauri default). Cosmetic only. Defer rename to
  v0.0.9.
- Settings tab schema_list has no drag-reorder yet. List + remove only.
  Defer drag to v0.0.9.
- Apple Event «event RimeRdpl» MAY not fire for IME-style apps (Smoodle
  is an input method, not a standard NSApplication). If `deploy_squirrel`
  errors, fallback: user clicks menubar S → Deploy manually after edits.
  Tauri returns the osascript stderr in the toast for diagnostics.
